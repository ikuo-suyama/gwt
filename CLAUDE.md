# Tools
- Use npm for package management
- Exec code: `npm run dev -- <args>` or `npm test`
- Build: `npm run build`

# Project Understanding
For first time, follow these interaction
- Read README.md and understand the project

# Design / Analysis / Requirements
- Do not simply comply with instructions; maintain a critical perspective.
- Make fact-based proposals, providing evidence and logical explanations.

# Code
- Obtain and understand the current time from the system date and time.
- If current branch is already merged into origin/master, create new branch on top of latest remote master
- Always strictly follow Kent Beck Style TDD.
- Adhere to the Single Responsibility Principle as much as possible and keep each class below 200 lines at max
- Make sure all tests pass before committing
- Update docs if we update related resources
- NEVER merge PR by yourself

# Directory Structure Management
- Follow the directory structure defined in CONTRIBUTING.md
- When adding new directories, update CONTRIBUTING.md's "Project Structure" section
